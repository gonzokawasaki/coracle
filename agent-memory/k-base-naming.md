The product name is **AMAdocs** — settled 2026-06-13 after explicitly weighing a rename. "LlamaDocs" was considered and **rejected** over trademark/positioning risk: it collides with **LlamaIndex** (a funded company/framework in the exact same LLM-document/RAG category — confusingly similar mark + identical market) and invites Meta's "Llama" brand-guideline scrutiny; it would also falsely imply a Llama-based product when AMAdocs deliberately avoids Llama-licensed models for its permissive-only (MIT/Apache) stance [[k-base-project]].

Open caveat (not blocking): "AMAdocs" is semi-descriptive ("ask my docs"), so it's weaker to protect and a clearance search (USPTO/EUIPO + app-store/domain) is still worth doing before public launch. Don't suggest renaming again unless the user reopens it.

**Why:** avoids redoing the naming debate and prevents me from proposing a rename later.
**How to apply:** use "AMAdocs" as the fixed product name; only revisit naming if the user raises it.
